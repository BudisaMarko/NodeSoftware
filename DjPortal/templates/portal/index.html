This is a test implementation. It consists of two parts that are run by independent web-applications:

  • The portal which is the VAMDC "central" that is supposed to let users make a data request without knowing about individual databases, to pass this request to the connected databases and to collect the results to be displayed or downloaded to the user. All URLs that contain /portal/ are run by this application.
  • The nodes which represent a data set, typically a database, with software running on top of it that allows communication with the portal. Nodes have URLs with /node/ and as you can see, there is currently just one: VALD.

Note: Even though both applications (/portal/ and /node/) run on the same server for test purposes, there is no "cheating" in the sense that they share running code. They only communicate via standard HTTP-interfaces which are important parts to be developed and tested.

Questions and feedback are welcome to thomas.marquart@fysast.uu.se.

For more details please visit the VAMDC-Wiki page NodeSoftware
